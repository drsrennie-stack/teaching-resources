iRAT & tRAT Specifications
TBL doctrine applied. RAT cognitive level is floored at DOK 2 (conceptual understanding). DOK 1 (recall only) and DOK 4 (full integration) are deliberately excluded: the RAT exists to verify students grasped the foundational concepts from prework, not to test recall and not to substitute for the application exercise.

Application & Facilitation
4S enforced. The application exercise is generated against the 4S framework (Significant problem, Same problem, Specific choice, Simultaneous reporting) with a defensible-disagreement requirement: the specific choice must be designed so reasonable, well-prepared teams could land on different answers and defend each. The AI is instructed to design the application FIRST, then build the RAT to scaffold the concepts it needs.

Export & Integration
What this does. When checked, the AI generates the standard TBL session AND adds a separate TSV block formatted for InteDashboard's bulk question upload template. The application exercise itself must still be set up manually in InteDashboard if it uses formats beyond MCQ, SATA, or free-response.

Template columns matched. The TSV output exactly matches the InteDashboard upload template column order: Question, Min Words, Max Words, Topics, Correct Answer, A, B, C, D, E. MCQ questions populate the option columns with a single Correct Answer letter. SATA questions use a comma-separated Correct Answer (e.g., "B,C"). Free-response questions populate Min Words and Max Words instead of options.
